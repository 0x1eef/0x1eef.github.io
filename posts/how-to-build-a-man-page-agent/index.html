About

This post documents how to build an agent that is specialized in answering queries from man pages. For this example we will implement one agent and two tools: a man page reader, and a man page searcher built on apropos. The agent will be supported by the builtin ActiveRecord support provided by llm.rb.

Migration

The first step is to define the table where the agent can be stored. For this case the built-in persistence contract is just one data column. This can be changed to jsonb where supported, but for simplicity we will use a text column:

create_table :agents do |t|
  t.text :data
  t.timestamps
end

Explanation

  • :data
    Stores the serialized agent runtime.
  • :timestamps
    Gives us the usual ActiveRecord created and updated timestamps.

Agent

The agent is implemented as an ActiveRecord model. It has instructions, a model, two tools, and a concurrency setting for tool execution:

require "llm"
require "active_record"
require "llm/active_record"

class Agent < ApplicationRecord
  acts_as_agent provider: :set_provider do
    model "gpt-5.4-mini"
    instructions "Answer questions from local UNIX man pages."
    tools Apropos, Man
    concurrency :thread
  end

  private

  def set_provider
    LLM.openai(key: ENV["OPENAI_SECRET"], persistent: true)
  end
end

Explanation

  • acts_as_agent provider: :set_provider
    Persists LLM::Agent state on the model and lets the record resolve its own provider.
  • model "gpt-5.4-mini" and instructions "..."
    Define the default model and system instructions for the agent.
  • tools Apropos, Man
    These are the two local tools the agent can call.
  • concurrency :thread
    Runs tool work with threads.

Tools

Apropos

require "shellwords"

class Apropos < LLM::Tool
  name "apropos"
  description "Search the man page index"
  parameter :query, String, "Query to search"
  required %i[query]

  def call(query:)
    output = `apropos #{query.shellescape}`
    matches = output.lines.map(&:chomp).reject(&:empty?).first(10)
    {query:, matches:}
  end
end

Explanation

  • parameter :query, String, ...
    Defines the query input for the tool.
  • required %i[query]
    Makes the query required.
  • `apropos #{query.shellescape}`
    Runs apropos and escapes the query before it reaches the shell.
  • first(10)
    Returns the first few matches.

Man

require "shellwords"

class Man < LLM::Tool
  name "man"
  description "Read a man page"
  parameter :page, String, "The man page to read, such as ls or printf"
  parameter :section, String, "The man page section, such as 1 or 5"
  required %i[page]

  def call(page:, section: nil)
    args = [section, page].compact.map(&:shellescape).join(" ")
    output = `MANPAGER=cat PAGER=cat MANWIDTH=80 man #{args}`
    text = output.gsub(/\x08./, "").strip
    {page:, section:, content: text[0, 12_000]}
  end
end

Explanation

  • parameter :page and parameter :section
    Defines the page input and the optional section input.
  • required %i[page]
    Makes the page required.
  • `MANPAGER=cat PAGER=cat MANWIDTH=80 man #{args}`
    Forces plain text output instead of opening a pager.
  • compact.map(&:shellescape).join(" ")
    Skips section when it is not given and escapes each argument.
  • output.gsub(/\x08./, "").strip
    Removes the backspace sequences that man sometimes uses for formatting, then trims the result.
  • text[0, 12_000]
    Returns a bounded slice of the page.

Conclusion

The example is small, but the pattern scales well. In this case the agent answers questions from local man pages instead of relying only on model training data.

That is what makes it useful. Once the model can search and read from a local source of truth, the same approach can be applied to other things like internal docs, config files, or logs.

The other useful part is that the agent is just a model with state and capabilities. In this post those capabilities are read-only, but the same pattern also works for agents that take action.

The other nice part is that the code stays small. The model owns the serialized runtime state, and the rest is ordinary ActiveRecord code:

##
# Create our agent
agent = Agent.create!

##
# First question - persists automatically
puts agent.talk("How do I extract a tar archive?").content

##
# Second question - persists automatically
puts agent.talk("What about gzipped tar archives?").content

##
# Third question - persists automatically
puts agent.talk("How do I block incoming traffic with pf.conf?").content
